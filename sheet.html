Setup & Config

First-Time Setup

git config --global user.name "Your Name"
Set the name attached to all your commits.
git config --global user.email "you@email.com"
Set the email attached to all your commits.
git config --global core.editor code
Set VS Code as the default editor. Use vim, nano, etc. as needed.
git config --list
View all current config settings.

Starting a Repo

git init
Turn the current folder into a Git repo. Creates a hidden .git directory.
git clone <url>
Copy a remote repo to your machine. Downloads all history and files.
git clone <url> folder-name
Clone into a specific folder name instead of the default repo name.
Daily Commands

Staging & Committing

git status
Show which files are modified, staged, or untracked. Run this constantly.
git add <file>
Stage a specific file for the next commit.
git add .
Stage all changes in the current directory and below.
git add -p
Interactively choose which changes to stage, chunk by chunk. Great for clean commits.
git commit -m "message"
Save staged changes to your local history with a message.
git commit -am "message"
Stage all tracked file changes and commit in one step. Skips git add for modified files. Does not include new untracked files.

Inspecting Changes

git diff
Show unstaged changes line by line.
git diff --staged
Show staged changes that are ready to commit.
git log
Show full commit history with author, date, and message.
git log --oneline --graph
Compact history view with a visual branch graph. Very useful.
git show <commit>
Show the diff and metadata for a specific commit.
git blame <file>
Show which commit last changed each line of a file, and who did it.

Undoing Things

git restore <file>
Discard unstaged changes in a file. Resets it to the last committed version.
git restore --staged <file>
Unstage a file without losing the changes in your working directory.
git commit --amend
Modify the most recent commit. Change the message or add forgotten files. Only use before pushing.
git revert <commit>
Create a new commit that undoes a previous one. Safe to use on shared branches.
git reset --soft HEAD~1
Undo the last commit but keep changes staged. Good for re-doing a commit.
git reset --hard HEAD~1
Undo the last commit and discard all changes permanently. Cannot be recovered easily.

Stashing

git stash
Save your uncommitted changes to a temporary stack and clean your working directory. Useful when you need to switch branches without committing.
git stash pop
Restore the most recently stashed changes and remove them from the stash stack.
git stash list
View all stashed entries with their index numbers.
git stash apply stash@{2}
Apply a specific stash by index without removing it from the stack.
git stash drop stash@{0}
Delete a specific stash entry.
Branches & Merging

Branches

git branch
List all local branches. An asterisk marks your current branch.
git branch <name>
Create a new branch at the current commit. Does not switch to it.
git switch <name>
Switch to an existing branch. Preferred over older git checkout.
git switch -c <name>
Create a new branch and switch to it in one command.
git branch -d <name>
Delete a branch. Git will warn you if it has unmerged commits. Use -D to force.
git branch -m old-name new-name
Rename a branch.

Merging & Rebasing

git merge <branch>
Merge the specified branch into your current branch. Creates a merge commit if histories diverge.
git merge --no-ff <branch>
Force a merge commit even when a fast-forward is possible. Preserves branch history visually.
git rebase <branch>
Replay your commits on top of another branch. Produces a linear, cleaner history. Do not rebase commits already pushed to a shared branch.
git rebase -i HEAD~3
Interactive rebase. Edit, squash, reorder, or drop the last 3 commits before pushing.
git cherry-pick <commit>
Apply a specific commit from another branch onto the current branch.

Resolving Conflicts

git status
After a merge conflict, this shows which files need resolution. Conflicted files are marked.
# Edit the conflicted file
Open the file and look for <<<<<<<, =======, and >>>>>>> markers. Keep what you want and remove the markers.
git add <resolved-file>
Mark a conflict as resolved by staging the file.
git merge --continue
Complete the merge after resolving all conflicts.
git merge --abort
Cancel the merge entirely and return to the pre-merge state.
Working with Remotes

Remotes

git remote -v
List all configured remotes with their URLs. origin is the default remote name.
git remote add origin <url>
Connect your local repo to a remote repo at the given URL.
git remote rename old new
Rename a remote.
git remote remove <name>
Remove a remote connection.

Push, Pull & Fetch

git push origin <branch>
Upload your local branch commits to the remote.
git push -u origin <branch>
Push and set the upstream tracking branch. After this, just use git push.
git push --force-with-lease
Force push safely. Fails if the remote has commits you have not fetched. Safer than --force.
git fetch
Download remote changes without merging them into your local branches. Safe to run anytime.
git pull
Fetch and merge remote changes into your current branch. Equivalent to fetch plus merge.
git pull --rebase
Fetch remote changes and rebase your local commits on top. Keeps history linear.
GitHub Workflow

Pull Requests

# 1. Create a feature branch
Always branch off main. Name it clearly: feature/login-form, fix/nav-bug.
# 2. Push the branch to GitHub
Use git push -u origin <branch>. GitHub will show a prompt to open a pull request.
# 3. Open a Pull Request
On GitHub, click "Compare & pull request". Write a clear title and description of what changed and why.
# 4. Review and merge
Reviewers comment, request changes, or approve. Once approved, merge using Squash, Merge, or Rebase depending on your team's convention.
# 5. Delete the branch
After merging, delete the remote branch on GitHub. Then run git branch -d <branch> locally.

Forking Workflow

# 1. Fork the repo on GitHub
Click Fork to create your own copy of a repo under your account. Used for contributing to projects you do not own.
git clone <your-fork-url>
Clone your fork locally.
git remote add upstream <original-url>
Add the original repo as a second remote called upstream. This lets you pull in updates from the source project.
git fetch upstream
Get new commits from the original project.
git merge upstream/main
Bring your local main up to date with the original project's main.

GitHub CLI (gh)

gh repo clone user/repo
Clone a GitHub repo using the CLI. Handles SSH setup automatically.
gh pr create
Open a pull request from your current branch via the terminal.
gh pr list
List open pull requests for the current repo.
gh pr checkout <number>
Check out a pull request locally by its number. Useful for reviewing others' work.
gh issue create
Create a new GitHub issue from the terminal.
gh repo fork --clone
Fork and clone a repo in one step.

.gitignore

# In .gitignore:
List file patterns that Git should never track. One pattern per line.
node_modules/
Ignore an entire folder. The trailing slash is recommended for directories.
*.log
Ignore all files with a .log extension using a wildcard.
.env
Ignore specific files. Always ignore files with secrets like API keys.
!important.log
The ! prefix negates a pattern. This re-includes a previously ignored file.
git rm --cached <file>
Stop tracking a file that is already committed. Add it to .gitignore first, then run this.
Advanced Commands

Tags & Releases

git tag v1.0.0
Create a lightweight tag at the current commit. Used to mark release points.
git tag -a v1.0.0 -m "Release"
Create an annotated tag with a message. Preferred for releases; stores extra metadata.
git push origin --tags
Push all local tags to the remote. Tags are not pushed automatically with git push.
git tag -d v1.0.0
Delete a local tag.

Searching History

git log --grep="keyword"
Search commit messages for a keyword.
git log -S "function_name"
Find commits that added or removed a specific string in the code. Useful for tracking when something changed.
git log -- <file>
Show the commit history for a single file.
git bisect start
Start a binary search through commit history to find when a bug was introduced. Mark commits as good or bad to narrow it down.

Submodules

git submodule add <url>
Embed another Git repo inside your repo as a submodule. The submodule tracks a specific commit of the dependency.
git submodule update --init
Initialize and fetch submodule content after cloning a repo that has submodules.
git clone --recurse-submodules <url>
Clone a repo and automatically initialize all its submodules.

Useful Aliases

git config --global alias.st status
Create git st as a shortcut for git status.
git config --global alias.co switch
Create git co as a shortcut for git switch.
git config --global alias.lg "log --oneline --graph"
Create git lg for a compact, visual log. One of the most useful aliases.
git config --global alias.undo "reset --soft HEAD~1"
Create git undo to quickly undo the last commit while keeping your changes staged.
Tips

Commit Messages

Write commit messages in imperative mood: "Add login form" not "Added login form". Keep the subject line under 72 characters. Reference issue numbers when relevant: "Fix null pointer in parser (#42)".

Reset vs Revert

Use git revert on branches others use. It adds a new commit. Use git reset only on local branches you have not pushed yet. Reset rewrites history.

Fetch vs Pull

git fetch downloads remote changes but does not touch your working files. Use it to see what changed before merging. git pull fetches and merges in one step.

HEAD Explained

HEAD is a pointer to your current position in the repo. Usually it points to the tip of your current branch. HEAD~1 means one commit before HEAD. HEAD~2 means two commits back.
